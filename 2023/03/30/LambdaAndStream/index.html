Lambda表达式

简化函数式接口的匿名内部类写法。

以三种方式得到比较器Comparator对象：


( 参数列表 ) -> { Lambda体 } ，得到实现了接口抽象方法的匿名类的实例对象，而无需new关键字。

Java8内置的四大核心函数式接口：

方法引用

若Lambda体中的内容有方法已经实现了，可以使用“方法引用”。Lambda表达式的另一种表现形式。

一个硬性前提：抽象方法的参数列表和返回值类型，必须与要使用方法引用的方法一致。

格式：

  • 对象引用::非静态方法名
  • 类名::静态方法名
  • 类名::非静态方法名

第三种格式的使用场景：
Lambda参数列表中的第一个参数是非静态方法的调用者，第二个参数是非静态方法的参数时，可以使用ClassName::MethodName(None-static)

构造器引用

调用与抽象方法参数列表相对应的那个构造器。

格式：
ClassName::new

显然，sp1 sp2 调用的是Employee的无参构造器，fun1 fun2 调用的是含有1个参数的构造器。

数组引用

格式：
Type[]::new

具体API

https://www.runoob.com/java/java8-streams.html

  1. 过滤：filter()

多级过滤执行顺序问题：

可见，对一个元素进行所有过滤操作后，迭代操作下一个元素。如果集合有序，迭代顺序与其一致。

  1. 切片：
    limit() 具有短路效益，提前过滤元素，减少不必要的操作。

  2. 去重：
    distinct()以对象的hashcode()和equals()方法进行去重。

  3. 映射：
    flatmap() 会把得到的每个元素的流合并成一个流。当应用map()，返回Stream<Stream<Object>>嵌套流结构时，应用flatmap()得到的是Stream<Object>。

  4. 排序：
    sorted(Comparator cmp)，无参数时为自然排序（按照Comparable<>接口的CompareTo()方法的具体实现来排序）。

  5. 返回单个元素：
    findFirst(),findAny(),min(),max()
    这些属于终止操作，并返回包装了元素的Optional<>容器，可以通过容器获得元素，也可以指定流返回null情形下容器的元素内容。

  6. reduce()对流中元素进行累积。
    T reduce(T identity, BinaryOperator<T> accumulator);第一个参数为累积初始值，作为二元运算的初始的第一个参数；第二个参数为二元运算操作（lambda表达式），运算结果为下一次迭代的二元运算操作的第一个参数，下次迭代元素为二元运算的第二个参数。
    累加示例：

    Optional<T> reduce(BinaryOperator<T> accumulator); 1参，返回Optional容器，因为没有设定初始值，返回结果可能为null，使用Optional对其封装。

  7. 收集：
    collect()接受一个Collector接口的实现，将流转换为其他形式，可以搭配Collectors工具类所提供的静态方法（返回函数式接口的实现）食用。
    转换流为任何集合：

    使用Stream实现基本数据类型数组转List

    其中，boxed() 打包，顾名思义，将基本数据类型的流转换为其包装类的流。IntStream对象调用boxed() 返回一个 Stream<Integer>对象。

Collectors.groupingBy(), Collectors.partitioningBy() 分组分区，得到map，多级分组分区，得到嵌套map。

Collectors.joining() 字符串拼接，返回拼接后的字符串。
可由参数指定连接符，前缀，后缀：public static Collector<CharSequence, ?, String> joining(CharSequence delimiter,CharSequence prefix,CharSequence suffix)

Collectors.summarizingInt() 得到一组样本的概要统计量对象，该对象封装了该组样本的常用统计量，如最值，均值等。


并行流与顺序流

parallel()：将当前的顺序流转换为并行流。底层使用ForkJoin模式。

sequential()：将当前的并行流转换为顺序流。
