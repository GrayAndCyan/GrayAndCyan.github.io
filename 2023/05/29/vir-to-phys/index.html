Linux内存虚实映射 - 根据进程pagemap获取物理地址并使用Devmem验证

操作系统课程设计任务之一，对指定变量进行内存虚实映射。本文更多的是题目完成记录与操作步骤描述，并无对Linux内核原理的进一步探讨。另外，OS课程设计的其他题目会在后续的文章中有所补充。

问题描述


strict alternation算法是更前面问题的内容，本文旨在记录和描述完成虚实映射任务的过程，strict alternation即严格替换算法是IPC（进程间通信）中使用忙等待实现互斥的一种算法，本文只使用了严格替换算法程序的 turn 变量，获取该变量的虚拟地址和物理地址，并使用devmem工具验证。


pagemap基本介绍


查看一个进程中的一个虚地址对应的实际物理地址，可以通过查看文件/proc/<pid>/pagemap 得到。其中pid是该进程的进程ID号。pagemap存放着这个进程的页表相关信息，其结构请参见下面一段资料：

计算物理地址


可以通过解析pagemap中的内容，得到一个进程虚地址所在页号对应的物理页框号（页框即上面资料所说的页帧（page frame），我更习惯叫做页框，OS老师课上是这样译的）。

那么具体做法如下：

  1. 得到变量虚拟地址，这点很好说，使用 & 操作符即可得到变量虚拟地址。
  2. 操作系统将内存分页，每个进程所占空间也分页存储在内存。一页大小通常是4KB，用变量的虚拟地址除以页面大小可以得到该地址所在的页号。
  3. 使用 lseek() 函数定位pagemap文件中上一步得到的虚拟页对应的64位值，使用 read() 读取这个值。
  4. 该64位值的高55位为页号对应的页框号，拿到页框号。
  5. turn变量的页内偏移即它在所在页框的偏移，使用虚拟地址对页面大小取余可以获得这个页偏移。
  6. 物理地址呼之欲出，即页框号乘页面大小（4KB），再加上上一步得到的页内偏移。

使用devmem验证物理地址正确性


我们可以查看物理地址上的内容是否和虚地址上的内容一致，验证上面方法得到的物理地址是否正确。如何查看物理地址上的内容？

我们可以使用devmem工具，该工具通过打开 /dev/mem 文件，并且将文件内容映射到一串地址上，通过对这些地址的访问达到对物理地址的访问。不过，为了能够让该工具能够访问/dev/mem文件，需要对内核进行一些配置：

通常内核是不允许访问0xfffff即1M空间以上的物理地址的。

解决方式：重新改编内核选项，重新下载编译安装内核，并且切换到这个内核上进行地址验证。编译安装内核的方法见我的上一篇文章 https://grayandcyan.github.io/2023/05/28/kernelproc/ 。在编译内核时，对于内核选项：

使用空格删除这一选项。最后通过vim检查 .config 文件，若出现 CONFIG_STRICT_DEVMEM is not set 则表示设置正确

同时，使CONFIG_X86_PAT=n，并开启 CONFIG_EXPERT 选项（即 CONFIG_EXPERT=y ，有的内核中可能没有该选项，此时忽略即可）。


功能测试与结果验证


编译完成后，对上述工作进行测试——获取虚实地址，并使用devmem验证物理地址上的值是否为 turn 变量的值。

先说下devmem用法：

address：要读取或写入的物理地址。以十六进制表示，例如 0x12345678。

type：操作类型。可选参数，表示要读取或写入的数据类型。默认为写入操作。可选值为 ‘b’（字节）、’h’（半字）和 ‘w’（字）。

data：要写入的数据。可选参数，以十六进制表示。

🔺很重要的注意事项：devmem 会对 /dev/mem 设备进行访问，它固然需要sudo权限，但计算物理地址的程序也不要忘记加上sudo权限，因为它需要读取pagemap文件，否则得到的物理地址只会是虚拟地址的后三位！！

先放测试结果，我的 turn 变量值为100，根据devmem得到的物理地址上的十六进制值是 0x64 ，是十进制的100，验证成功。另外，turn 为 int 型，4字节，所以devmem类型参数为h，半字。
