Linux下内核模块实现的设备驱动编程

操作系统课程设计任务之一，通过内核模块方式实现的设备驱动编程。本文更多的是题目完成记录与操作步骤描述，并无对Linux内核原理的进一步探讨。希望对同学们理解OS课设任务与解决相关问题有所帮助。

问题描述


内核模块

概念

内核模块是一段可以动态加入到内核，成为内核一部分的代码。在运行中扩展内核的功能而不需要重新编译内核。因此本题将设备驱动程序作为内核模块，实现更灵活省时的动态加载。

编写rwbuf.c

一个模块至少需要入口函数和退出函数：

  1. 入口函数由module_init()宏指定，在模块被加载的时候被调用。入口函数的返回值为0表示成功，非0表示失败。

  2. 退出函数由module_exit()宏指定，在模块被卸载时被调用，主要来完成资源的清理工作，它被调用完毕后，该模块就被内核清除了。

这些宏都在头文件linux/module.h定义。使用这些宏只是用来提供识别信息。


printk() 函数:

printk() 函数在Linux内核中定义并且对模块可用，为内核提供日志功能， 记录内核信息或用来给出警告。与标准 C 库函数 printf 的行为相似。信息添加到文件 /var/log/kern.log ，可直接查看，或用命令dmesg查看，当然，使用管理员权限。


内核编程时的注意事项：

  • 内核编程时不能访问C库
  • 内核编程时必须使用GNU C
  • 内核编程时缺乏像用用户空间那样的内存机制
  • 内核编程时浮点数很难使用
  • 内核只有一个很小的定长堆栈
  • 由于内核支持异步中断、抢占和SMP，因此必须时刻注意同步和并发
  • 要考虑可移植性的重要性

编写Makefile

在 Linux 内核模块的编译过程中，使用 gcc 命令单独编译是不够的，因为内核模块的编译需要与内核的编译环境和规则相匹配。

Makefile 是一个构建工具，它可以根据预定义的规则和依赖关系自动化构建过程。在 Linux 内核模块开发中，Makefile 用于将内核模块源代码与内核源代码进行编译和链接，以生成最终的模块文件（.ko 文件）。

Makefile 文件中的命令会调用适当的编译器（如 gcc）以及内核构建系统提供的工具链和规则，以确保内核模块的正确编译和链接。这包括正确的编译选项、头文件路径、符号解析和模块加载等。

因此，在 Linux 内核模块开发中，使用 make 工具结合 Makefile 文件是一种常见且推荐的编译方式，以确保模块与内核环境的兼容性和正确性。

VFS管理字符设备接口： struct file_operations

该结构中重要的函数接口: open，read，write，release，ioctl

我们要做的是在设备驱动程序中实现上面这几个函数。

rwbuf 是我们的设备名，根据函数名便能看出，rwbuf_open(), rwbuf_close() 分别在打开和关闭设备时调用，rwbuf_read(), rwbuf_write() 分别在读和写设备时调用，rwbuf_ioctl 用于清除和模式切换。

这些函数不需要用户程序主动调用，在用户程序中，对设备文件调用系统调用函数 open，read，write，release，ioctl 进行操作时，内核会根据 rwbuf_fops 自动映射到我们在驱动程序中编写的相应的实现，并调用这些函数实现。


rwbuf.c 完整代码如下，下面对其中的实现细节进行解释：

  1. inuse 全局变量是一种锁机制，保证不能同时打开多个该设备。当设备打开时，inuse 值为1， 此时其他程序再试图打开设备时失败会报错。

  2. 字符指针类型的 rwbuf 和 last_rwbuf 分别保存当前字符串和修改前的上一次字符串内容。当 rwbuf 在 rwbuf_write() 中被修改前，会使用 memcpy() 进行内存拷贝，将内容拷贝到 last_rwbuf 中。memcpy() 是定义在<linux/string.h> 头文件的函数， 这里没有显式引用该头文件是因为它包含在<linux/kernel.h> 和 <linux/module.h> （但这不利于代码可读性）。

  3. rwbuf_ioctl() 接收来自形参变量 cmd 的命令，通过更改全局变量 mod 实现三种模式的切换： 清除/读上次/读本次。在读的时候程序会对当前 mod 进行条件判断。

    🔺注意， ioctl 的 cmd 不能等于 2 ， 所以在驱动程序的ioctl实现中， 避免用到数值为2的命令定义。如下第二行， 我将原本的“读上次”命令 RW_READ_LAST 的宏定义从 2 改成了 22 。

  4. mmap 实现：

    • 简介：在驱动编程中，mmap（内存映射）是一种机制，用于将设备驱动程序中的内存映射到用户空间，使用户可以直接访问设备内存，而无需进行显式的读写操作。

    • 实现步骤：

      1. 验证映射请求的合法性，包括映射的大小和权限等检查。
      2. 将设备内存的物理地址转换为页帧号（PFN）。
      3. 调用remap_pfn_range函数将页帧号映射到用户空间的虚拟地址范围。
      4. 返回映射结果，成功返回0，失败返回错误码（在上面实现中我是偷懒返回的-1）。

    • virt_to_phys()函数将rwbuf的虚拟地址转换为物理地址，并将其右移PAGE_SHIFT位，得到物理页框号。PAGE_SHIFT 值为 12，即一页的通常大小 4KB 等于 2^12 字节。

    • kzalloc() 函数是 Linux 内核提供的一个内存分配函数，用于在内核空间中分配内存，并将分配的内存块初始化为零。它的函数原型如下：

      kzalloc 函数的作用是分配一块大小为 size 字节的内存，并将该内存块的所有字节初始化为零。与 kmalloc 函数不同，kzalloc 在分配内存后会自动初始化为零，而无需显式调用其他函数或手动循环遍历来清零。这个函数适用于在内核中需要分配内存并初始化为零的场景，比如分配结构体、缓冲区或其他数据结构。

      这里我在模块初始化函数中，使用 kzalloc() 为 rwbuf 和 last_rwbuf 分配内存。
